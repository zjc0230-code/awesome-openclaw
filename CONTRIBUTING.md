# 贡献指南 🤝

感谢您对 Awesome OpenClaw 项目的关注！我们欢迎各种形式的贡献。

---

## 🎯 贡献方式

### 1. 推荐新的 Skill

如果您发现了优秀的 OpenClaw Skill，欢迎推荐！

#### 提交流程

1. **Fork 本仓库**
2. **填写 Skill 推荐表单** (使用 [templates/skill-recommendation.md](templates/skill-recommendation.md))
3. **实际测试该 Skill**
4. **撰写详细评测** (至少 200 字)
5. **提交 Pull Request**

#### Skill 推荐标准

必须满足以下所有条件：

- ✅ **功能性**: 实际可用，非玩具项目
- ✅ **安全性**: 通过 VirusTotal 检测，无恶意代码
- ✅ **文档**: 有清晰的使用文档
- ✅ **维护**: 最近 6 个月有更新
- ✅ **许可证**: 使用开源许可证

加分项：

- ⭐ 有中文文档
- ⭐ 适配国内网络环境
- ⭐ 提供实战案例
- ⭐ GitHub stars > 100

---

### 2. 改进现有文档

我们需要：

- 📝 修正拼写和语法错误
- 📝 补充使用案例和示例代码
- 📝 翻译英文文档为中文
- 📝 优化排版和格式

#### 修改流程

1. **直接编辑** Markdown 文件
2. **提交 Pull Request**
3. **说明修改理由**

---

### 3. 报告问题

发现任何问题都可以提 Issue：

- 🐛 **Bug 报告**: Skill 失效、安全问题
- 💡 **功能建议**: 新的分类、改进建议
- ❓ **使用问题**: 安装、配置相关问题

#### Issue 模板

```markdown
## 问题类型
- [ ] Bug 报告
- [ ] 功能建议
- [ ] 使用问题

## 详细描述
[请详细描述问题]

## 复现步骤 (如果是 Bug)
1. ...
2. ...

## 期望行为
[描述您期望的结果]

## 环境信息
- OpenClaw 版本: 
- 操作系统: 
- 相关 Skill: 
```

---

### 4. 贡献代码

我们欢迎以下类型的代码贡献：

- 🛠️ **自动化脚本**: 自动检查 Skills 有效性
- 🛠️ **CI/CD**: GitHub Actions 工作流
- 🛠️ **工具**: 辅助开发和测试的工具

#### 代码规范

- 使用 **Python 3.8+** 或 **Bash**
- 遵循 **PEP 8** (Python) 或 **Google Shell Style Guide** (Bash)
- 必须包含 **注释** 和 **使用说明**
- 通过 **测试** (如果适用)

## ✅ Pull Request 检查清单

提交 PR 前请确认：

- [ ] 已阅读并遵循本贡献指南
- [ ] 代码/文档遵循项目规范
- [ ] 已实际测试相关功能
- [ ] PR 标题清晰描述了改动内容
- [ ] 填写了 PR 描述模板
- [ ] 如推荐新 Skill，已填写完整评测报告
- [ ] 如修改代码，已添加注释和文档

---

## 🔍 审查流程

### PR 审查标准

1. **功能性**: 是否符合项目定位
2. **质量**: 内容是否准确、完整
3. **格式**: 是否符合 Markdown 规范
4. **原创性**: 是否有抄袭或重复内容

### 审查时间

- 文档修改: **24-48 小时**
- Skill 推荐: **3-5 个工作日** (需要人工测试)
- 代码贡献: **5-7 个工作日**

### 审查结果

- ✅ **通过**: 直接合并
- 🔄 **需要修改**: 会留下具体的修改建议
- ❌ **拒绝**: 说明拒绝理由
