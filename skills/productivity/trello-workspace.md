## 🚀 快速开始

### 📋 前置条件

1. **Trello 账号**（免费或付费版本均可）
2. **OpenClaw 环境**（Node.js ≥18.0）
3. **Trello API 密钥**（从 [Trello App Key](https://trello.com/app-key) 获取）

---

### 🔧 安装配置

#### **步骤 1：获取 Trello API 凭证**

```bash
# 1. 访问 https://trello.com/app-key
# 2. 获取 API Key（24 位字符串）
# 3. 点击 Token 链接生成 Token（64 位字符串）
# 4. 可选：复制你的主看板 ID（从看板 URL 中提取）
#    示例：https://trello.com/b/BOARD_ID/board-name
```

#### **步骤 2：安装 Trello MCP 服务器**

```bash
# 方式 1：npm 全局安装
npm install -g @xenral/trello-mcp

# 方式 2：Docker 安装（推荐生产环境）
docker pull xenral/trello-mcp:latest
docker run -d \
  -e TRELLO_API_KEY="your_api_key" \
  -e TRELLO_TOKEN="your_token" \
  -e TRELLO_BOARD_ID="default_board_id" \
  -p 3000:3000 \
  xenral/trello-mcp
```

#### **步骤 3：配置 OpenClaw**

在 `~/.openclaw/config.json` 中添加 Trello MCP 服务器：

```json
{
  "mcpServers": {
    "trello": {
      "command": "npx",
      "args": ["-y", "@xenral/trello-mcp"],
      "env": {
        "TRELLO_API_KEY": "your_trello_api_key_here",
        "TRELLO_TOKEN": "your_trello_token_here",
        "TRELLO_BOARD_ID": "your_default_board_id" // 可选
      }
    }
  }
}
```

#### **步骤 4：重启 OpenClaw 并测试**

```bash
# 重启 OpenClaw
openclaw restart

# 测试连接
openclaw ask "列出我的所有 Trello 看板"

# 预期输出：
# ✅ 找到 3 个看板：
#   1. 个人工作区 (ID: abc123)
#   2. 团队项目 (ID: def456)
#   3. 产品规划 (ID: ghi789)
```

## 🎯 实战场景

### **场景 1：敏捷开发团队的 Sprint 管理**

**背景**：10 人开发团队，每两周一个 Sprint，需高效管理 Backlog、Sprint、Code Review、Done 流程。

**传统方式痛点**：
- 每日站会手动整理任务进度（15 分钟）
- Sprint 开始时手动拖拽任务到 Sprint 列表（30 分钟）
- Code Review 状态需频繁手动更新（每天 10+ 次）

**使用 Trello Workspace Skill 解决方案**：

```bash
# 1. Sprint 开始：批量移动优先级任务
"将所有标记为'P0'和'P1'的卡片从 Backlog 移到'Sprint 42'"

# 2. 每日站会自动化
"生成昨天已完成、今天进行中、遇到阻塞的任务汇总"
→ AI 自动分析并生成 Slack 消息

# 3. GitHub 集成：PR 自动更新
# 在 GitHub Actions 配置 Webhook
"当 PR 被批准时，自动将对应的 Trello 卡片移到'待部署'列表"

# 4. Sprint 结束：数据统计
"统计本 Sprint 完成的卡片数量、未完成原因、平均完成时间"
→ 输出：完成 32/35 任务，3 个任务因依赖阻塞延期，平均 2.3 天完成

# 效率提升：
# - 每日站会时间：15 分钟 → 5 分钟（节省 67%）
# - Sprint 规划时间：30 分钟 → 3 分钟（节省 90%）
# - 任务状态同步：手动 10+ 次/天 → 自动化（节省 100%）
```
